# Бот мини-игр (Telegram)

Один файл `minigames_bot.py` на aiogram 3 (SQLite) — казино-мини-игры с
внутриигровой бесплатной валютой и секретной админ-панелью только для
владельца.

## Файлы

- `minigames_bot.py` — весь код бота.
- `requirements.txt` — зависимости (нужен `aiogram>=3.29.0` для цветных
  кнопок и premium-эмодзи, Bot API 9.4).
- `.env.example`, `Procfile`, `.gitignore`.

## Команды / триггеры (пишутся прямо в чат, не команды)

- `рулетка <сумма>` — например `рулетка 100кк`
- `ракетка <сумма>` — краш-игра, кнопка «Забрать» в любой момент
- `мины <сумма>` — поле 6×6, 5 бомб, x1.5 за клетку, кнопка «Забрать»
- `майн <сумма>` или `майн макс` — поле 5×5: 5 бомб, алмаз (x5), незерит
  (x10), железо (x1.98), медь (x1.58), камень — нейтральная клетка
- `пер <сумма>` (ответом на сообщение игрока) — перевод
- `кнб <сумма>` (ответом на сообщение игрока) — камень-ножницы-бумага,
  соперник принимает/отклоняет, дальше оба выбирают ход
- `ежедневка` / `бонус` — бесплатный бонус, кулдаун настраивается
  (по умолчанию раз в час)
- `баланс` / `balance`
- `хелп` / `help` / `помощь`
- `/buy` — VIP-статус за Stars (см. ниже)

## Про VIP и Stars — важное ограничение

Я осознанно НЕ стал делать продажу игровой валюты за Stars — платить
реальные деньги за фишки, которые потом ставятся в игру со случайным
исходом, юридически является азартной игрой на реальные деньги и прямо
запрещено правилами Telegram для ботов.

Вместо этого `/buy` продаёт VIP-статус как обычную косм—etику/QoL:
- бейдж 👑 в игровых сообщениях
- множитель к ежедневному бонусу (настраивается в админке)

VIP никак не меняет шансы и множители в самих играх.

## Про цвета и premium-эмодзи

Bot API 9.4 (9 февраля 2026) добавил у `InlineKeyboardButton` поля `style`
(`primary`/`success`/`danger` — синий/зелёный/красный) и
`icon_custom_emoji_id`. Для эмодзи нужна Telegram Premium у владельца
бота (аккаунта, на который зарегистрирован токен) — иначе Telegram не
покажет иконку. Всё это настраивается в скрытой админ-панели: доступна
только владельцу (`OWNER_ID`) и только в личных сообщениях с ботом —
никто другой её не увидит и не откроет команду `/admin`.

## Админ-панель (`/admin`, только в личке, только OWNER_ID)

- **Баланс** — найти игрока по ID, выдать/списать любую сумму (в т.ч.
  себе)
- **Бан/разбан**
- **Тексты** — редактирование всех сообщений бота с плейсхолдерами
  ({currency}, {amount}, {multiplier} и т.д.)
- **Кнопки** — подпись, цвет, premium-эмодзи для каждой кнопки бота
- **Настройки** — название валюты, множители рулетки, шаг мин,
  сумма/кулдаун бонуса, цена и бонус VIP, макс. множитель ракетки,
  стартовый баланс
- **Клетки Майна** — emoji/premium-эмодзи, множитель и количество для
  каждого типа клетки (бомба, алмаз, незерит, железо, медь)
- **Статистика** — число игроков, админов и т.д.
- **Логи ошибок** — последние необработанные исключения бота, чтобы
  сразу видеть, что сломалось, не копаясь в консоли сервера

## Надёжность / нагрузка

- Все операции с БД асинхронные (aiosqlite), включён `PRAGMA
  journal_mode=WAL` для параллельного доступа.
- Активные игры (ракетка/мины/майн/кнб) хранятся в памяти процесса —
  быстро, без лишних запросов к БД на каждый тик анимации. Минус: при
  перезапуске бота активные партии обнуляются (ставка не сгорает —
  она уже списана, но и не доиграна; это ограничение любого
  in-memory подхода, для полной устойчивости к рестартам нужно
  переносить состояние игр в Redis/БД — могу сделать отдельным
  апгрейдом, если понадобится).
- Глобальный обработчик ошибок ловит все необработанные исключения,
  пишет их в БД (видно в админке → Логи ошибок) и не даёт всему
  процессу упасть из-за одной сломавшейся игры.
- Для по-настоящему высокой нагрузки (тысячи сообщений в секунду)
  рекомендуется в будущем перейти с SQLite на PostgreSQL — сама
  архитектура кода это допускает без переписывания игровой логики.

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env   # и заполнить
export BOT_TOKEN="токен_от_BotFather"
export OWNER_ID="твой_telegram_id"
python minigames_bot.py
```

Своей Telegram ID можно узнать у @userinfobot.
